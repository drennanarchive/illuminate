Roadmap phases

A practical path from “current best version” to a more polished, magical, distinct desktop/mobile experience.

1

Polish the current build

Lowest riskNext iteration
Build
  • Richer foreground sparkles without raising total particle count too much.
  • Low / medium / high sparkle setting.
  • More distinct status panel text and icons.
  • Refined reset animation while preserving reset guards.
Success looks like

It feels prettier and more responsive, but not meaningfully heavier. Reset remains boringly reliable.

2

Make the constellation meaningful

High delightCore magic
Build
  • Each completed checklist item lights one star or orb.
  • A tiny star trail flies from card to constellation.
  • Lines connect between lit stars.
  • Completion draws or brightens the full constellation.
Success looks like

The checklist stops feeling like five boxes and starts feeling like a tiny sky slowly coming alive.

3

5

Consider a technical upgrade

Only if neededPerformance gate
Possible upgrade
  • Canvas-based particles if DOM particles become limiting.
  • Config object for all copy, themes, and checklist items.
  • Optional PWA shell later if we want app-like installability.
Rule

Do not add complexity just because it is clever. Upgrade only when it directly improves smoothness, maintainability, or the magical feel.

Decision points

Questions to answer before the next substantial implementation round.

Visual density

How sparkly should “default” be?

Recommended: make medium the default, with low/high sparkle modes available. High can be dramatic; medium should remain bedtime-friendly.

Humor

How visible should Portal references be?

Recommended: rare and optional. Think easter egg, not theme. The primary identity is Starlight, not Aperture.

Scope

Should checklist items become editable?

Recommended: later. First polish the fixed experience. Personalization is valuable, but it can dilute the curated ritual if added too early.

Tech

DOM effects or Canvas?

Recommended: stay DOM/CSS for the next pass. Consider Canvas only if richer effects become laggy on mobile.

Testing scorecard

Use this after each new version to decide whether it is truly better.

Ask after every build
  • Does it feel laggy on phone?
  • Does desktop feel meaningfully different from mobile?
  • Do the sparkles feel foreground enough?
  • Does reset clear reliably?
  • Does reset accidentally trigger completion?
  • Does the status panel feel distinct?
  • Do the list cards feel alive?
  • Are the sounds magical or too sharp/loud?
  • Does it still feel bedtime-friendly?
