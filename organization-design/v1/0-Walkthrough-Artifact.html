Leadership Operating Rhythm Package

How the artifacts fit together and when to use each one
The big picture:

These artifacts are not three versions of the same thing. They operate at three different levels:

Operating Model → Engagement Design → Execution Tool

Together, they answer:
1. How should we use leadership time?
2. What should the leadership touchpoint accomplish?
3. What information should we bring each month?

Artifact 3: Leadership Operating Rhythm Map

Highest-level view | "How we work together"

Purpose

Defines the overall cadence between the team and leadership. It prevents:
  • Random status requests
  • Unclear escalation paths
  • Overloading leadership calendars

Primary User

Leadership + team leads Used when shaping:
  • Meeting cadence
  • Information flow
  • Roles and expectations

Question It Answers

"How do we intentionally engage leadership?"

Use When

  • Leader role changes
  • Team is newly elevated
  • Cadence needs redesign

Artifact 1: Monthly Leadership Touchpoint Concept

Decision point | "Does this forum make sense?"

Purpose

Introduces the concept of the monthly PMR-inspired discussion. It helps leadership react before implementation.

Primary User

Government lead / senior stakeholder

Question It Answers

"Would this be a valuable use of my time?"

Use When

  • Seeking buy-in
  • Introducing the idea
  • Aligning expectations

Artifact 2: Monthly PMR Lite Pre-Read

Execution tool | "What do we discuss each month?"

Purpose

The recurring product used to prepare for the monthly leadership touchpoint.

Primary User

Team members / workstream owners

Question It Answers

"What does leadership need to know this month?"

Use When

  • Preparing monthly review
  • Capturing changes
  • Preparing decisions

Recommended Implementation Sequence

Step 1: Use Artifact 3 internally
Define how leadership engagement should work.

Step 2: Review Artifact 1 with the lead
Validate the monthly touchpoint concept.

Step 3: Build Artifact 2 after alignment
Create the recurring monthly pre-read.

Important Guardrails

Avoid allowing the PMR to become:
  • A task tracker
  • A project status meeting
  • A duplicate of operational syncs
  • A reporting burden
The value comes from:
  • Strategic context
  • Decision support
  • Leadership awareness
  • Priority alignment
Core idea: A small governance team does not need more reporting. It needs a predictable mechanism for information flow, alignment, and leadership engagement.
